# Design Guidelines

Updated on `2026-06-01`

## Direction

The visual system should move further toward a lightweight archival interface inspired by `architectuurgids.nl`, while staying cleaner and more international in tone.

## Visual Rules

- Background: white
- Text: black
- Supporting lines: light gray or soft black only
- Accent use: minimal and rare
- No dark glass, no heavy gradients, no cinematic framing
- No portal-like widgets or dense dashboard panels

## Layout Rules

- Left navigation remains the core archive pattern on desktop
- Mobile uses a reduced top rail
- The homepage is an entry page, not a campaign page
- Buildings appear in readable sequence
- The map is important but secondary
- Lists should read like editorial indexes, not cards first

## Typography

- Use a restrained sans-serif for the main reading experience
- Display type can exist, but should be quiet and sparse
- Large titles should feel like section markers, not branding slogans
- Korean and English should both remain readable without visual imbalance

## Interaction

- Fast initial load matters more than decorative motion
- Any animation should be slow, sparse, and structural
- Hover states should clarify, not dramatize
- Filters must remain understandable at a glance

## What To Avoid

- heavy map-first landing pages
- large multi-panel dashboards
- full-screen hero messaging
- oversized card walls
- too many colors
- too many badges visible at once
- institutional portal density

## Map Guidance

- The map should support the archive, not dominate it
- Map views should be available as one route among several
- On the homepage, featured buildings should stay stronger than the map
- If the map appears on a mixed page, it should not push the reading list below the fold unnecessarily

## Award Metadata Guidance

- Awards should not become a top-level visual identity
- Award information should appear inside the building detail metadata
- If a project won or was associated with `젊은건축가상`, surface that in the detail view or project facts

## International Readability

- Every important archive label should support Korean and English
- Korean projects should default to Korean display when the UI language is Korean
- English mode should feel intentional, not machine-translated or half-localized

## Design Reference Summary

Keep from `architectuurgids.nl`:

- archive entrance logic
- left rail navigation
- immediate category access
- low-friction reading

Do not copy from heavier portals:

- multi-layered control surfaces
- GIS-first framing
- crowded category blocks
- heavy institutional chrome
